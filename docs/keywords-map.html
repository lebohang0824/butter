Keywords Map

A technical analysis of Butter's attention-weighted vocabulary, signal-to-noise efficiency, and logic gate mapping for LLM reasoning.

Butter's keyword system is designed with a deep understanding of how large language models allocate attention. Each keyword carries a specific semantic weight that influences how an LLM interprets, prioritises, and executes specification instructions. This page maps those weights, compares Butter's signal density against JSON and natural language, and visualises how keywords act as deterministic logic gates that prune hallucination branches.

Attention Scaling

Large language models do not treat all tokens equally. Structural keywords — words that define what to do, when to do it, and what must be true — receive disproportionate attention during inference. Butter's vocabulary is engineered to exploit this: every keyword is chosen to maximise semantic signal per token.

Semantic Scaling Impact

Butter's keyword weights follow a deliberate hierarchy: execution keywords (action, feature) carry the highest attention because they define what the AI must build. Constraint keywords (enforce, if, unless) follow closely because they gate execution paths. Declaration keywords (param, type, required) sit in the middle because they shape the input schema. Metadata keywords (app, description, version) carry the least weight because they provide context without driving behaviour.

This hierarchy means an LLM reading a Butter spec allocates its reasoning budget proportionally: most attention goes to execution steps, then to constraints, then to type declarations, and least to metadata. The result is fewer hallucinated actions, stricter condition evaluation, and more type-safe implementations — all without explicit prompting instructions.

Signal-to-Noise Efficiency

Semantic density measures how much actionable information a format delivers per token. Butter DSL is engineered for maximum signal density: every token either declares intent, defines a constraint, or specifies execution order. JSON and natural language carry significant structural overhead — brackets, quotes, keys, filler words — that consumes context window tokens without adding execution value.

Logic Gate Map

Butter keywords function as deterministic logic gates in an LLM's reasoning pipeline. Each keyword either declares an action node (what must happen) or acts as a modifier gate (under what conditions it happens). Together, they form a decision tree that prunes non-deterministic hallucination branches before they form.

How It Works

When an LLM reads a Butter spec, it processes the keyword hierarchy top-down:

  1. feature — Opens a scope. The LLM knows everything inside belongs to one capability.
  2. action — Creates a concrete execution node. The LLM adds this to its ordered task list.
  3. enforce — Attaches a hard constraint to the preceding action. The LLM must verify this before proceeding.
  4. if / unless / when / while — Gates the action behind a condition. The LLM evaluates the predicate and decides whether to execute or skip.

Without these gates, an LLM faced with "process payments" might invent its own conditions, skip validation steps, or execute actions in parallel. Butter's keyword gates eliminate this by making every branching decision explicit and every execution order deterministic.

Pruning Impact

Every Butter keyword acts as a filter. Without if/unless/when/while gates, an LLM must decide on its own which actions apply — and will frequently invent conditions or execute actions that should be skipped. With Butter's explicit gates:

  • if prunes actions when the predicate is false — the LLM skips instead of hallucinating execution.
  • unless prunes actions when the predicate is true — the LLM skips the inverse path.
  • when prunes actions until the event fires — the LLM waits instead of assuming immediate execution.
  • while prunes actions when the state condition is false — the LLM recognises the action is state-dependent.
  • enforce prunes incomplete execution — the LLM cannot proceed until the constraint is satisfied.

The result is a specification where every execution path is deterministic. The LLM doesn't need to infer what to do — it reads what to do, evaluates the gates, and executes only the validated paths. This is the core mechanism behind Butter's claim of up to 100% alignment in a single shot.
